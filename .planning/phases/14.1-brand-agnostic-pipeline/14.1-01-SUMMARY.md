# Phase 14.1 Plan 01: Delete SKILL_FILES/loadStagePrompt Infrastructure Summary

**One-liner:** Replaced file-system skill-loading with a synchronous `buildSystemPrompt()` that tells agents to query DB tools for brand context at runtime.

## What Was Built

- **Deleted infrastructure:** `SKILL_FILES` constant, `getFallbackPrompt()`, `STAGE_HEADING_PATTERNS`, `extractStageSection()`, `loadStagePrompt()` ŌĆö all removed from `api-pipeline.ts`
- **New function:** `buildSystemPrompt(stage, ctx)` ŌĆö synchronous, exported, no disk reads, returns generic process instructions with tool list per stage
- **Updated `runStageWithTools`:** calls `buildSystemPrompt` directly (no `await`)
- **Brand references purged:** `buildCopyPrompt`, `buildLayoutPrompt`, `buildStylingPrompt`, `buildSpecCheckPrompt`, `buildFixPrompt` all updated ŌĆö "Fluid" removed from all prompt text
- **`PipelineContext` extended:** added optional `brandName?: string` field
- **Tests updated:** `loadStagePrompt` describe block replaced with 8 `buildSystemPrompt` tests; file-system tests updated to use `CLAUDE.md` and `tools/` directory instead of deleted `brand/` directory

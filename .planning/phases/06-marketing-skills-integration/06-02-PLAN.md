**A. Extend sync.sh with marketing skills distribution:**

Add a new section AFTER the existing Cursor install block and BEFORE the REPORT section. This section copies marketing skills to ~/.agents/skills/ (overwrite):

```bash
# ──────────────────────────────────────────────
# INSTALL: MARKETING SKILLS (global distribution)
# ──────────────────────────────────────────────
MARKETING_SKILLS_DIR="$REPO_DIR/skills/marketing"

if [ -d "$MARKETING_SKILLS_DIR" ]; then
  echo ""
  echo "--- Marketing Skills (global) ---"

  for skill_dir in "$MARKETING_SKILLS_DIR"/*/; do
    skill_name=$(basename "$skill_dir")
    dst_dir="$HOME/.agents/skills/$skill_name"

    if [ ! -f "$skill_dir/SKILL.md" ]; then
      echo "  SKIP: $skill_name (SKILL.md not found)"
      continue
    fi

    run mkdir -p "$dst_dir"

    if [ "$DRY_RUN" = true ]; then
      echo "  [dry-run] cp -r $skill_dir/* $dst_dir/"
    else
      cp -r "$skill_dir"/* "$dst_dir/"
    fi

    echo "  Overwrote: $dst_dir"
    ((INSTALLED++))
  done
fi
```

Use cp -r to handle skills with references/ subdirectories. Overwrite is intentional per locked decision (repo is authoritative source).

Also update the uninstall section to handle marketing skills cleanup (remove skills/marketing/* from ~/.agents/skills/ during --uninstall).

Update the help text to mention marketing skills distribution.

Update the REPORT section verify command to include: `echo "  ls ~/.agents/skills/ | grep -c ''"` for marketing skill count.

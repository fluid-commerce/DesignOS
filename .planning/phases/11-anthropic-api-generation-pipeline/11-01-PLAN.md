**Anthropic client initialization:**
- Import `Anthropic` from `@anthropic-ai/sdk`
- Load env file using `process.loadEnvFile(path.resolve(__dirname, '../../.env'))` wrapped in try/catch (Node 24 native — no dotenv)

**Pipeline types:**
```typescript
export type PipelineStage = 'copy' | 'layout' | 'styling' | 'spec-check';
export type FixTarget = 'copy' | 'layout' | 'styling';

export interface StageResult {
  stage: PipelineStage;
  output: string; // text output from the stage
  toolCalls: number; // how many tool loops ran
}

export interface PipelineContext {
  prompt: string;
  creationType: string; // 'instagram' | 'linkedin' | 'one-pager'
  workingDir: string; // absolute path to working directory for this creation
  htmlOutputPath: string; // absolute path where final HTML goes
  creationId: string;
  campaignId: string;
}
```

**Stage model mapping (per user decision: haiku for layout, sonnet for creative):**

**Tool schemas** — define 4 tools as `Anthropic.Tool` objects:
- `read_file`: takes `{ path: string }`. Description: "Read a file from disk. Use to load brand docs, patterns, templates, or generated files."
- `write_file`: takes `{ path: string, content: string }`. Description: "Write content to a file on disk."
- `list_files`: takes `{ directory: string, pattern?: string }`. Description: "List files in a directory."
- `run_brand_check`: takes `{ html_path: string }`. Description: "Run brand-compliance validation on an HTML file. Returns JSON result with pass/fail and issues."

Export these as `PIPELINE_TOOLS` (all 4) and subsets per stage:
- Copy/Layout/Styling: `read_file`, `write_file`, `list_files`
- Spec-check: `read_file`, `write_file`, `run_brand_check`

**Tool executor** — `executeTool(name: string, input: Record<string, unknown>, workingDir: string): Promise<string>`:
- `read_file`: Resolve path relative to project root if not absolute. Read with `fs.readFile(resolvedPath, 'utf-8')`. Return file contents. On error, return error message string (not throw — tool results should be strings).
- `write_file`: Resolve path. **Security check**: verify resolved path starts with workingDir (prevent writes outside sandbox). Write with `fs.writeFile`. Return "File written: {path}".
- `list_files`: Resolve directory. Read with `fs.readdir`. If `pattern` provided, filter with simple glob match. Return newline-joined list.
- `run_brand_check`: Resolve html_path. Run `node tools/brand-compliance.cjs {resolved_path}` via `execSync`. Return stdout (JSON result). On error, return stderr.
- Unknown tool: return "Unknown tool: {name}"

The project root is derived as `path.resolve(__dirname, '../..')` (two levels up from `canvas/src/server/`).

**Stage prompt loader** — `loadStagePrompt(stage: PipelineStage, ctx: PipelineContext): string`:
Returns a system prompt string for each stage. These are hardcoded prompt templates (not reading skill .md files) that capture the delegation logic from `fluid-social.md`:

- **copy**: "You are a Fluid brand copywriter. Generate marketing copy for a {ctx.creationType} social post. Use the read_file tool to load brand/voice-rules.md and brand/social-post-specs.md for brand voice guidance. Write a structured copy file (headline, subtext, accent color, archetype) to {ctx.workingDir}/copy.md."
- **layout**: "You are a Fluid layout agent. Create structural HTML layout for a {ctx.creationType} post. Use read_file to load brand/layout-archetypes.md. Read the copy from {ctx.workingDir}/copy.md. Write layout HTML to {ctx.workingDir}/layout.html."
- **styling**: "You are a Fluid styling agent. Apply Fluid brand styling to the layout. Use read_file to load brand/design-tokens.md, brand/asset-usage.md, and patterns/index.html. Read {ctx.workingDir}/copy.md and {ctx.workingDir}/layout.html. Write complete styled HTML to {ctx.htmlOutputPath}."
- **spec-check**: "You are a Fluid spec-check agent. Validate the HTML at {ctx.htmlOutputPath}. Run brand-compliance validation using the run_brand_check tool. Read brand/social-post-specs.md for spec reference. Write a JSON report to {ctx.workingDir}/spec-report.json with { overall: 'pass'|'fail', blocking_issues: [...] }."

Each prompt should also include the creation dimensions and reference to the brand/ directory path.

Export `loadStagePrompt` so Plan 02 can call it.

**SSE emission helpers** — export functions that match the NDJSON format stream-parser.ts already handles:

```typescript
export function emitStageStatus(res: ServerResponse, creationId: string, stage: string, status: string): void
```
Each wraps `res.write()` in try/catch. Format must match what stream-parser.ts parses (see interfaces block above).

**Anthropic client initialization:**
- Import `Anthropic` from `@anthropic-ai/sdk`
- Load env file using `process.loadEnvFile(path.resolve(__dirname, '../../.env'))` wrapped in try/catch (Node 24 native — no dotenv)
- Export a singleton `anthropic` client instance. The SDK auto-reads `ANTHROPIC_API_KEY` from process.env.

**Pipeline types:**
```typescript
export type PipelineStage = 'copy' | 'layout' | 'styling' | 'spec-check';
export type FixTarget = 'copy' | 'layout' | 'styling';

export interface StageResult {
  stage: PipelineStage;
  output: string; // text output from the stage
  toolCalls: number; // how many tool loops ran
}

export interface PipelineContext {
  prompt: string;
  creationType: string; // 'instagram' | 'linkedin' | 'one-pager'
  workingDir: string; // absolute path to working directory for this creation
  htmlOutputPath: string; // absolute path where final HTML goes
  creationId: string;
  campaignId: string;
}
```

**Stage model mapping (per user decision: haiku for layout, sonnet for creative):**
```typescript
export const STAGE_MODELS: Record<PipelineStage, string> = {
  'copy': 'claude-sonnet-4-5-20250514',
  'layout': 'claude-haiku-4-5-20250514',
  'styling': 'claude-sonnet-4-5-20250514',
  'spec-check': 'claude-sonnet-4-5-20250514',
};
```

**Tool schemas** — define 4 tools as `Anthropic.Tool` objects:
- `read_file`: takes `{ path: string }`. Description: "Read a file from disk. Use to load brand docs, patterns, templates, or generated files."
- `write_file`: takes `{ path: string, content: string }`. Description: "Write content to a file on disk."
- `list_files`: takes `{ directory: string, pattern?: string }`. Description: "List files in a directory."
- `run_brand_check`: takes `{ html_path: string }`. Description: "Run brand-compliance validation on an HTML file. Returns JSON result with pass/fail and issues."

Export these as `PIPELINE_TOOLS` (all 4) and subsets per stage:
- Copy/Layout/Styling: `read_file`, `write_file`, `list_files`
- Spec-check: `read_file`, `write_file`, `run_brand_check`

**Tool executor** — `executeTool(name: string, input: Record<string, unknown>, workingDir: string): Promise<string>`:
- `read_file`: Resolve path relative to project root if not absolute. Read with `fs.readFile(resolvedPath, 'utf-8')`. Return file contents. On error, return error message string (not throw — tool results should be strings).
- `write_file`: Resolve path. **Security check**: verify resolved path starts with workingDir (prevent writes outside sandbox). Write with `fs.writeFile`. Return "File written: {path}".
- `list_files`: Resolve directory. Read with `fs.readdir`. If `pattern` provided, filter with simple glob match. Return newline-joined list.
- `run_brand_check`: Resolve html_path. Run `node tools/brand-compliance.cjs {resolved_path}` via `execSync`. Return stdout (JSON result). On error, return stderr.
- Unknown tool: return "Unknown tool: {name}"

The project root is derived as `path.resolve(__dirname, '../..')` (two levels up from `canvas/src/server/`).

**Stage prompt loader** — `async loadStagePrompt(stage: PipelineStage, ctx: PipelineContext): Promise<string>`:

Per user decision: "Existing skill .md files become system prompts for API calls — same files, different loading mechanism — server reads from disk and injects into API call." This keeps the read path swappable for Phase 14 (DB-backed brand intelligence).

Implementation:

1. Define a mapping from `creationType` to skill file path:
```typescript
const SKILL_FILES: Record<string, string> = {
  'instagram': path.join(os.homedir(), '.agents/skills/fluid-social/SKILL.md'),
  'linkedin': path.join(os.homedir(), '.agents/skills/fluid-social/SKILL.md'),
  'one-pager': path.join(os.homedir(), '.agents/skills/fluid-one-pager/SKILL.md'),
  'theme-section': path.join(os.homedir(), '.agents/skills/fluid-theme-section/SKILL.md'),
};
```

2. Read the skill file from disk using `fs.readFile(skillPath, 'utf-8')`. The skill files contain the full delegation messages for each stage (copy, layout, styling, spec-check sections).

3. Extract the relevant stage section from the loaded skill content. The skill files use markdown headings to delimit stages (e.g., `## Copy Agent`, `## Layout Agent`, etc.). Parse the loaded content to extract the section matching the current `stage`. Use a regex like `/## (Copy|Layout|Styling|Spec.?Check).*?\n([\s\S]*?)(?=\n## |\n---|\Z)/i` to pull the relevant block.

4. Compose the final system prompt by combining:
   - A brief role preamble: "You are a Fluid {stage} agent working on a {ctx.creationType} creation."
   - The extracted section content from the skill file (the bulk of the prompt)
   - Context variables injected: `ctx.workingDir`, `ctx.htmlOutputPath`, `ctx.creationType`
   - Available tool references so the agent knows what tools it has

5. **Fallback**: If the skill file read fails (file not found, parse error), log a warning and fall back to a hardcoded minimal prompt for that stage. The fallback prompts are:
   - **copy**: "You are a Fluid brand copywriter. Generate marketing copy for a {ctx.creationType} post. Use read_file to load brand/voice-rules.md and brand/social-post-specs.md. Write copy to {ctx.workingDir}/copy.md."
   - **layout**: "You are a Fluid layout agent. Create HTML layout for a {ctx.creationType} post. Use read_file to load brand/layout-archetypes.md. Read copy from {ctx.workingDir}/copy.md. Write layout to {ctx.workingDir}/layout.html."
   - **styling**: "You are a Fluid styling agent. Apply brand styling. Use read_file to load brand/design-tokens.md, brand/asset-usage.md, patterns/index.html. Read {ctx.workingDir}/copy.md and {ctx.workingDir}/layout.html. Write styled HTML to {ctx.htmlOutputPath}."
   - **spec-check**: "You are a Fluid spec-check agent. Validate {ctx.htmlOutputPath}. Use run_brand_check tool. Write report to {ctx.workingDir}/spec-report.json."
   These fallbacks are last-resort only — the primary path MUST read from disk.

6. Export `loadStagePrompt` so Plan 02 can call it. Note: this is now `async` (returns `Promise<string>`) since it does file I/O.

**SSE emission helpers** — export functions that match the NDJSON format stream-parser.ts already handles:

**4. Add POST /api/dam-sync endpoint in the middleware chain, before the existing /api/brand-assets handler:**

```typescript
// POST /api/dam-sync — trigger manual DAM sync
if (url === '/api/dam-sync' && method === 'POST') {
  const token = process.env.VITE_FLUID_DAM_TOKEN;
  if (!token) {
    res.writeHead(400, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ error: 'VITE_FLUID_DAM_TOKEN not configured' }));
    return;
  }
  try {
    const result = await runDamSync(token, path.join(projectRoot, 'assets'));
    res.writeHead(200, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(result));
  } catch (err) {
    res.writeHead(500, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ error: String(err) }));
  }
  return;
}
```

**On mount DAM detection:**
1. Fetch brand assets
2. If any asset has `source === 'dam'`, infer DAM is configured. Show sync bar with "Synced on startup" initially.
3. If none have source=dam, attempt a lightweight probe: POST /api/dam-sync. If 400, set status to 'no-token' (hide bar). If 200 or error, show bar (DAM configured but maybe empty/errored).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Button padding vertical, inline element gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding (matches AssetsScreen `1.5rem`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Page-level top padding (not used in this phase) |

Exceptions:
- Asset card badge: 4px 8px (xs/sm) — tight inline label

Resolved non-multiples (checker fixes applied):
- Sync status bar: `8px 12px` padding (was 10px 12px — 10 replaced with 8, nearest multiple of 4)
- "Sync now" button: `8px 16px` padding (was 8px 14px — 14 replaced with 16, nearest multiple of 4)

**Source:** AssetsScreen.tsx measured values, index.css reset.

## Component Inventory

New UI surface introduced in Phase 13 is minimal. All new elements integrate into the existing AssetsScreen viewport.

### 1. Sync Status Bar

Location: top of AssetsScreen, below the page heading row, above the asset grid.

Anatomy:
- Icon: cloud sync SVG (inline, 16x16, strokeWidth 1.75)
- Text: "Last synced [relative time]" or "Syncing…" or "DAM unreachable — showing cached assets"
- "Sync now" button (right-aligned)

Padding: `8px 12px` (multiples of 4).

States:
- **idle**: `#555` text (`--text-muted`). "Last synced 3 minutes ago." Button: accent bg `#44B2FF`, text `#0d0d0d`, 6px border-radius.
- **syncing**: accent spinner (CSS `@keyframes spin`), "Syncing…" text in `#888` (`--text-secondary`). Button disabled, opacity 0.5.
- **error**: `rgba(200,80,80,0.15)` background, `#e88` text. "DAM unreachable — showing cached assets. [Sync now]" link in accent.
- **no-token**: hidden entirely (DAM token not configured).

### 2. "Removed from DAM" Badge

Location: overlaid on top-right corner of asset card (absolute position) when `dam_deleted = 1`.

Anatomy: pill label, amber tint, 4px 8px padding, 13px font-size (label size), weight 500.

Style: `background: rgba(255,180,50,0.15); color: #cc9; border-radius: 999px; font-size: 13px; font-weight: 500; padding: 4px 8px`.

No action required from the user — purely informational. Card remains in grid. Tooltip on hover: "This asset was removed from the Fluid DAM Brand Elements folder. You can delete it manually."

### 3. DAM Source Badge (optional, on card)

Location: bottom-left of asset card thumbnail, small pill.

Anatomy: "DAM" text, 13px (label size), weight 500, `#44B2FF` at 70% opacity text, transparent background.

Visible only on `source = 'dam'` assets. Local assets show nothing.

### 4. "Sync now" Button

Padding: `8px 16px` (multiples of 4). Background: `#44B2FF` (accent). Text: `#0d0d0d`. Border-radius: 6px. Font-size: 13px (label), weight 500.

## Interaction Contract

### Sync Flow

1. On app startup: `runDamSync()` fires fire-and-forget. Sync status bar shows "Syncing brand assets…" while in progress.
2. On completion: status bar updates to "Last synced just now".
3. On error (DAM unreachable): status bar transitions to error state. No blocking modal. No alert. Existing cached assets remain visible.
4. "Sync now" button: POST `/api/dam-sync`. Button shows spinner (replaces icon) during request. On response, refresh asset list via existing `fetchAssets()` pattern.
5. Assets with `dam_deleted = 1`: remain in grid with amber badge. User removes manually via existing delete flow.

### Token Not Configured

When `VITE_FLUID_DAM_TOKEN` is absent:
- Sync status bar is hidden entirely.
- Assets tab shows only local assets (no DAM-specific UI).
- No warning banner (would be noise for users who don't use DAM).

### No New Navigation

Phase 13 does not add nav tabs, modals, or new viewports. All changes surface within the existing Assets tab (AssetsScreen).

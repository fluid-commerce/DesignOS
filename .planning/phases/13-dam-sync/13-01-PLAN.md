**2. Create canvas/src/server/dam-client.ts:**

This module provides typed fetch wrappers for the Fluid DAM REST API. Server-only, never import from React.

Constants:
- `DAM_BASE_URL = 'https://api.fluid.app/api'`

Functions to export:

`getCompanyIdFromToken(token: string): number` — decode JWT payload (no signature verification needed server-side): `JSON.parse(Buffer.from(token.split('.')[1], 'base64').toString()).company_id`. Return the company_id number.

`isRawDamAsset(value: unknown): value is RawDamAsset` — type guard checking `typeof value === 'object' && value !== null && 'code' in value && 'name' in value && 'variants' in value && Array.isArray((value as Record<string, unknown>).variants)`.

`flattenDamTree(tree: Record<string, unknown>, depth = 0): RawDamAsset[]` — recursively walk the tree object. If depth > 10, return []. For each value in Object.values(tree): if isRawDamAsset, push it; else if object, recurse. Return flat array.

`damQuery(token: string, params: { path: string; cursor?: string; limit?: number }): Promise<DamQueryResponse>` — POST to `${DAM_BASE_URL}/dam/query` with headers `Authorization: Bearer ${token}`, `Content-Type: application/json`, `x-fluid-client: admin`. Body: JSON.stringify(params). Return parsed JSON as `{ tree: Record<string, unknown>; meta?: { next_cursor?: string } }`.

`listBrandElements(token: string, companyId: number, folderName = 'brand_elements'): Promise<RawDamAsset[]>` — paginated wrapper. Path: `${companyId}.${folderName}.*`. Loop calling damQuery with cursor until meta.next_cursor is falsy. Flatten each page's tree and concat results.

Export the `RawDamAsset` interface:
```typescript
export interface RawDamAsset {
  id: number;
  code: string;
  name: string;
  category: string;
  canonical_path: string;
  default_variant_url?: string | null;
  updated_at: string;
  variants?: Array<{ id: string; mime_type: string; processing_status: string }>;
}
```

**1. Create canvas/src/server/dam-sync.ts:**

Server-only module. Imports: listBrandElements, getCompanyIdFromToken, RawDamAsset from ./dam-client. upsertDamAsset, softDeleteRemovedDamAssets, DamAssetRow from ./db-api. nanoid, fs/promises, path.

Helper `sanitizeFilename(name: string): string` — replace non-alphanumeric chars (except dots and hyphens) with underscores; lowercase.

Helper `getMimeTypeFromUrl(url: string): string` — extract extension from URL path, map to mime type using same switch as asset-scanner.ts (.png, .jpg, .jpeg, .svg, .webp, .gif, .ttf, .woff, .woff2, .otf). Default 'application/octet-stream'.

`downloadAsset(url: string, destPath: string): Promise<number>` — fetch the CDN URL (no auth needed for default_variant_url). Convert response to arrayBuffer, write to destPath via fs.writeFile(destPath, Buffer.from(arrayBuffer)). Return byte count (arrayBuffer.byteLength). Create parent directory with fs.mkdir(dirname, {recursive: true}) before writing.

`runDamSync(token: string, assetsDir: string): Promise<DamSyncResult>` — the main sync function:
1. Decode companyId from token via getCompanyIdFromToken(token)
2. Call listBrandElements(token, companyId) to get all assets
3. Create `assets/brand/dam/` subdirectory under assetsDir if needed
4. For each asset with a truthy default_variant_url:
   - Compute local filename: `${asset.code}-${sanitizeFilename(asset.name)}${extFromUrl}`
   - Compute local filePath relative to assetsDir: `brand/dam/${filename}` (this is what goes in DB file_path, consistent with asset-scanner pattern where file_path is relative)
   - Wait, the existing brand_assets file_path is relative like `brushstrokes/brush-01.png`. The Vite middleware serves from the project `assets/` dir at `/fluid-assets/`. So for DAM assets, file_path should be `dam/${filename}` if we put them in `assets/dam/` OR `brand/dam/${filename}` if nested. Actually looking at asset-scanner, it scans `assets/` dir and treats top-level subdirectories as categories. So put DAM files at `{assetsDir}/dam/{filename}` and file_path = `dam/${filename}`.

   Actually wait — the assetsDir passed to scanAndSeedBrandAssets is `path.join(projectRoot, 'assets')`. The existing files live at e.g. `assets/brushstrokes/brush-01.png` with file_path `brushstrokes/brush-01.png`. So for DAM files, create `{assetsDir}/dam/` subdirectory. file_path = `dam/${filename}`. Full disk path = `path.join(assetsDir, 'dam', filename)`.
   - Download file to full disk path
   - Build DamAssetRow: damId=asset.code, name=asset.name, category='dam' (or asset.category from DAM), filePath=`dam/${filename}`, mimeType from URL extension, sizeBytes from download, damUrl=asset.default_variant_url, damModifiedAt=asset.updated_at
   - Call upsertDamAsset(row)
5. Collect all asset.code values into a Set. Call softDeleteRemovedDamAssets(codeSet).
6. Return `{ synced: number, skipped: number, softDeleted: number, errors: string[] }`

Export the `DamSyncResult` interface:
```typescript
export interface DamSyncResult {
  synced: number;
  skipped: number;
  softDeleted: number;
  errors: string[];
}
```

Wrap the entire function body in try-catch. On network errors, log warning and return result with error message. Never throw — caller should not need to handle exceptions.

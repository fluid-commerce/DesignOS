copy.md format:
```markdown
# Copy Output
## Platform: instagram|linkedin
## Accent: orange|blue|green|purple
## Archetype: problem-first|quote|app-highlight|partner-alert|stat-proof|manifesto|feature-spotlight
### HEADLINE
### BODY
### TAGLINE
### CTA
### SIDE_LABEL
```

layout.html format:
- Structural HTML with positioned containers
- `<!-- SLOT: name -->` comments marking content positions
- CSS class hooks: `fluid-[archetype]-[element]`
- `<!-- target: WxH -->` dimension comment
- No inline styles (deferred to styling agent)

styled.html format:
- Complete self-contained HTML/CSS
- @font-face declarations embedded
- All brand tokens applied (colors, spacing, opacity)
- Brushstrokes, circles, footer positioned
- Assets referenced via paths relative to project root

spec-report.json format:
```json
{
  "status": "pass|fail",
  "checks": {
    "deterministic": { "brand-compliance": {...}, "dimensions": {...} },
    "holistic": { "layout-balance": {...}, "copy-tone": {...}, ... }
  },
  "blocking_issues": [{ "source": "...", "severity": N, "issue": "...", "fix_target": "copy|layout|styling" }],
  "warnings": [...],
  "overall": "pass|fail"
}
```

- `maxTurns: 15`

**Contract (document in comment block):**
- INPUTS: platform (instagram/linkedin), path to styled.html, accent color, archetype
- OUTPUTS: .fluid-working/spec-report.json
- MAX_ITERATIONS: 1 (spec-check runs once; orchestrator handles re-runs)

**Step 1: Deterministic Checks**
Run CLI tools via Bash on `.fluid-working/styled.html`:
- `node tools/brand-compliance.cjs .fluid-working/styled.html --context social` — parse JSON stdout
- `node tools/dimension-check.cjs .fluid-working/styled.html --target <platform>` — parse JSON stdout
Map CLI output to the spec-report.json `deterministic` section.

**Step 2: Holistic Review**
Read the styled.html file and evaluate against brand docs:
- **layout-balance**: Does it match the expected archetype? (Read layout-archetypes.md for reference)
- **copy-tone**: Does text sound like Fluid brand voice? (Read voice-rules.md)
- **visual-hierarchy**: Is the headline dominant? Not information-dense?
- **brushstroke-placement**: 2 per post, edge-bleed, variety, screen blend mode, 0.10-0.25 opacity?
- **circle-sketch-usage**: Emphasis-only (wrapping specific words), mask-image approach (not hue-rotate)?
- **footer-structure**: Flag icon + We-Commerce wordmark left, Fluid dots right, correct padding?
- **accent-color-consistency**: Single accent color throughout?
- **font-usage**: flfontbold for impact, Inter for body? No substitutions?

For each holistic check: pass/fail, severity (using brand weight system 1-100), description of issue if fail, and `fix_target` (copy, layout, or styling) indicating which subagent should fix it.

**Step 3: Write Report**
Write JSON to `.fluid-working/spec-report.json` with structure:
```json
{
  "status": "pass|fail",
  "checks": {
    "deterministic": {
      "brand-compliance": { "errors": N, "warnings": N, "details": [...] },

    },
    "holistic": {
      "layout-balance": { "pass": bool, "severity": N, "note": "...", "fix_target": "layout" },
      "copy-tone": { "pass": bool, "severity": N, "note": "...", "fix_target": "copy" },
      "visual-hierarchy": { "pass": bool, "severity": N, "note": "...", "fix_target": "styling" },
      "brushstroke-placement": { "pass": bool, "severity": N, "note": "...", "fix_target": "styling" },
      "circle-sketch-usage": { "pass": bool, "severity": N, "note": "...", "fix_target": "styling" },
      "footer-structure": { "pass": bool, "severity": N, "note": "...", "fix_target": "styling" },
      "accent-color-consistency": { "pass": bool, "severity": N, "note": "...", "fix_target": "styling" },
```

Create the orchestrator skill at `.claude/skills/fluid-social/SKILL.md`.

**YAML frontmatter:**
```yaml
---
name: fluid-social
description: "Generate brand-correct Fluid social posts from a simple prompt. Creates Instagram (1080x1080) or LinkedIn (1200x627/1340x630) posts with the full brand treatment -- copy, layout, styling, and spec-check."
context: fork
disable-model-invocation: true
argument-hint: '"topic or brief" [--platform instagram|linkedin] [--product connect|payments] [--template archetype] [--variations N] [--ref path] [--debug]'
allowed-tools: Agent, Bash, Read, Write, Glob, Grep, Edit
---
```

**Skill body -- the orchestrator prompt (this IS the prompt, not a document):**

**Section 1: Argument Parsing**
Parse `$ARGUMENTS` for:
- Main prompt: everything that is not a flag (the topic/brief)
- `--platform`: instagram (default) or linkedin. Instagram = 1080x1080. LinkedIn = 1200x627 or 1340x630.
- `--product`: connect or payments (optional). When provided, gives the copy agent product-specific context so it can reference the correct Fluid product features, terminology, and pain points. If omitted, the copy agent infers product from the prompt content (or writes product-agnostic copy).
- `--template`: archetype name. Valid: quote, app-highlight, partner-alert, problem-first, stat-proof, manifesto, feature-spotlight. Also support natural language ("use the pain post template" -> problem-first). When specified, subagents follow template structure closely.
- `--variations N`: number of distinct takes to generate (default: 1). Each variation runs the full pipeline. Use numbered subdirectories: `.fluid-working/v1/`, `.fluid-working/v2/`, etc.
- `--ref path/to/file.html`: explicit reference file. Also support natural language references ("make it like the 3AM server fire post").
- `--debug`: preserve `.fluid-working/` directory after completion.

For natural language template matching: read `templates/social/index.html`, scan description text for each template, match against user's prompt terms.

**Section 2: Working Directory Setup**
- Create `.fluid-working/` directory (or `.fluid-working/v1/` etc. for variations)
- If `.fluid-working/` already exists, warn and create a new one with timestamp suffix

**Section 3: Pipeline Execution**
For each variation (default: 1):

Step 3a -- Copy Agent:
- Delegate to `copy-agent` via Agent tool
- Delegation message: "Generate Fluid brand copy for a social post. Topic: {prompt}. Platform: {platform}. {If product: Product context: {product} (use product-specific features, terminology, and pain points).} {If template: Follow the structure of templates/social/{template}.html closely.} {If ref: Reference the style of {ref}.} Write output to .fluid-working/copy.md"
- Wait for completion
- Print status: "Copy complete -- accent: {color}, archetype: {archetype}" (read from .fluid-working/copy.md)

Step 3b -- Layout Agent:
- Delegate to `layout-agent` via Agent tool
- Delegation message: "Create structural HTML layout for a Fluid social post. Platform: {platform}. Read copy from .fluid-working/copy.md. {If template: Follow the layout structure of templates/social/{template}.html closely.} Write output to .fluid-working/layout.html"
- Wait for completion
- Print status: "Layout complete -- archetype: {archetype}"

Step 3c -- Styling Agent:
- Delegate to `styling-agent` via Agent tool
- Delegation message: "Apply Fluid brand styling to the layout. Platform: {platform}. Read copy from .fluid-working/copy.md (for accent color). Read layout from .fluid-working/layout.html. Reference patterns/index.html for brand building blocks (footer, brushstrokes, circles). {If template: Match the visual styling of templates/social/{template}.html.} Write complete self-contained HTML to .fluid-working/styled.html"
- Wait for completion
- Print status: "Styling complete"

Step 3d -- Spec-Check Agent:
- Delegate to `spec-check-agent` via Agent tool
- Delegation message: "Validate the Fluid social post. Platform: {platform}. Accent color: {color from copy.md}. Archetype: {archetype from copy.md}. Read .fluid-working/styled.html. Run CLI tools and holistic review. Write report to .fluid-working/spec-report.json"
- Wait for completion
- Read .fluid-working/spec-report.json
- Print status: "Spec-check complete -- {pass/fail}"

**Section 4: Fix Loop**
If spec-report.json `overall` is "fail":

For iteration 1 to 3:
  1. Read `blocking_issues` from spec-report.json
  2. Group issues by `fix_target` (copy, layout, styling)
  3. For each fix target, re-delegate to that agent with feedback:
     - Delegation message: "FIX ITERATION {N}: Re-read the current .fluid-working/{agent-output-file}. The following issues were found: {issues list with severity and description}. Fix these issues and rewrite the output file."
  4. If any copy fixes were applied, cascade: re-run layout-agent, then styling-agent (this counts as one iteration, not three)
  5. Re-run spec-check-agent
  6. If `overall` is "pass", break the loop
  7. Print status: "Fix iteration {N} complete -- {pass/fail}"

If 3 iterations exhausted and still failing:
- Print escalation message with remaining issues
- Copy best attempt to output anyway, marked as draft

**Section 5: Output and Cleanup**
- Copy `.fluid-working/styled.html` (or `final.html` after fixes) to `./output/`
- Naming: `./output/fluid-social-{platform}-{archetype}-{timestamp}.html`
- For variations: `./output/fluid-social-{platform}-{archetype}-v{N}-{timestamp}.html`
- If NOT `--debug`: delete `.fluid-working/` directory
- If `--debug`: print "Debug: working directory preserved at .fluid-working/"
- Print final status: "Social post saved to: {output path}"

**Section 6: Status Reporting Format**
Throughout execution, print clear status updates:
```
Generating Fluid social post...
  Platform: instagram (1080x1080)
  Product: connect (or: inferred from prompt)
  Template: (none / problem-first / etc.)

[1/4] Copy...        done (accent: orange, archetype: problem-first)
[2/4] Layout...      done
[3/4] Styling...     done
[4/4] Spec-check...  pass

Saved: ./output/fluid-social-instagram-problem-first-20260310.html
```

**Anti-patterns to explicitly avoid (include these warnings in the skill body):**
- NEVER pass file contents in Agent delegation messages -- always reference by path
- NEVER let a subagent try to use the Agent tool -- only the orchestrator delegates
- NEVER reference assets from Reference/ -- use assets/ paths
- NEVER use hue-rotate for circle recoloring -- use mask-image + backgroundColor
- NEVER load all brand docs -- each agent loads only its contracted 2-4 docs

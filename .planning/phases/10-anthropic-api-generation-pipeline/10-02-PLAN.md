**`runStageWithTools` — the core agentic loop for one pipeline stage:**

```typescript
export async function runStageWithTools(
  stage: PipelineStage,
  userPrompt: string,
  ctx: PipelineContext,
  res: ServerResponse,
): Promise<StageResult>
```

Implementation:
1. Get system prompt via `loadStagePrompt(stage, ctx)`
2. Get model via `STAGE_MODELS[stage]`
3. Get tools subset: spec-check gets `[read_file, write_file, run_brand_check]`, others get `[read_file, write_file, list_files]`
4. Initialize `messages: Anthropic.MessageParam[] = [{ role: 'user', content: userPrompt }]`
5. Emit stage start: `emitStageStatus(res, ctx.creationId, stage, 'starting')`
6. Enter agentic loop (max 20 iterations as safety):
   - Call `anthropic.messages.create({ model, system, max_tokens: 8192, tools, messages, stream: false })`
   - For each `block` in `response.content`:
     - If `block.type === 'text'`: accumulate text, call `emitText(res, ctx.creationId, block.text)`
     - If `block.type === 'tool_use'`: call `emitToolStart(res, ctx.creationId, block.name)`
   - If `response.stop_reason === 'end_turn'`: break loop
   - If `response.stop_reason === 'tool_use'`:
     - Execute each tool_use block: `await executeTool(block.name, block.input, ctx.workingDir)`
     - Emit `emitToolDone` for each
     - Push assistant turn: `messages.push({ role: 'assistant', content: response.content })`
     - Push tool results: `messages.push({ role: 'user', content: toolResults })`
     - Continue loop
   - If `response.stop_reason === 'max_tokens'`: emit error status, break
7. Emit stage done: `emitStageStatus(res, ctx.creationId, stage, 'done')`
8. Return `{ stage, output: accumulatedText, toolCalls }`

**`runApiPipeline` — the full 4-stage pipeline for one creation:**

```typescript
export async function runApiPipeline(
  ctx: PipelineContext,
  res: ServerResponse,
): Promise<void>
```

Implementation:
1. Create working directory: `await fs.mkdir(ctx.workingDir, { recursive: true })`
2. Run stages sequentially:
   - **Copy stage**: `runStageWithTools('copy', buildCopyPrompt(ctx), ctx, res)`
     - `buildCopyPrompt` includes: the user's prompt, creation type, dimensions, instruction to write copy.md
   - **Layout stage**: `runStageWithTools('layout', buildLayoutPrompt(ctx), ctx, res)`
     - Includes: creation type, instruction to read copy.md, write layout.html
   - **Styling stage**: `runStageWithTools('styling', buildStylingPrompt(ctx), ctx, res)`
     - Includes: instruction to read copy.md + layout.html, write to htmlOutputPath
   - **Spec-check stage**: `runStageWithTools('spec-check', buildSpecCheckPrompt(ctx), ctx, res)`
     - Includes: instruction to validate htmlOutputPath, write spec-report.json

3. After spec-check, read `{workingDir}/spec-report.json`:
   - Try `JSON.parse(await fs.readFile(...))`, default to `{ overall: 'pass' }` on read error
   - If `overall === 'pass'`: done
   - If `overall === 'fail'`: enter fix loop

4. **Fix loop** (max 3 iterations):
   - Read `blocking_issues` from spec-report.json
   - Group by `fix_target` (copy, layout, styling)
   - For each group, run the fix agent: `runStageWithTools(target, buildFixPrompt(target, issues, ctx), ctx, res)`
   - **Cascade rule**: if 'copy' was in the fix targets, also re-run layout and styling (even if they had no issues)
   - Re-run spec-check: `runStageWithTools('spec-check', buildSpecCheckPrompt(ctx), ctx, res)`
   - Re-read spec-report.json. If pass, break.
   - After 3 iterations, emit warning and continue (best-effort output)

5. The fix loop should emit stage statuses like `emitStageStatus(res, ctx.creationId, 'fix-1', 'starting')` etc.

**Build prompt helpers** (private functions):
- `buildCopyPrompt(ctx)`: "Generate Fluid brand copy for a {ctx.creationType} marketing creation. Topic: {ctx.prompt}. Write structured copy (headline, subtext, accent color, archetype selection) to {ctx.workingDir}/copy.md."
- `buildLayoutPrompt(ctx)`: "Create structural HTML layout for a Fluid {ctx.creationType} post. Read copy from {ctx.workingDir}/copy.md. Write layout to {ctx.workingDir}/layout.html."
- `buildStylingPrompt(ctx)`: "Apply Fluid brand styling. Read copy from {ctx.workingDir}/copy.md, layout from {ctx.workingDir}/layout.html. Write complete self-contained HTML to {ctx.htmlOutputPath}."
- `buildSpecCheckPrompt(ctx)`: "Validate the HTML at {ctx.htmlOutputPath}. Use run_brand_check tool. Write JSON report to {ctx.workingDir}/spec-report.json with format { overall: 'pass'|'fail', blocking_issues: [{ description, severity, fix_target }] }."
- `buildFixPrompt(target, issues, ctx)`: "FIX: Re-read and fix the following issues in your domain. Issues: {JSON.stringify(issues)}. Rewrite the relevant file."

Note: Keep prompts concise for now. User explicitly wants iterative approach — these can be refined after testing.

```typescript
else {
  // ── API MODE: Anthropic SDK pipeline ─────────────────────
  if (activeCampaignGeneration !== null) {
    res.writeHead(409, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ error: 'Campaign generation already in progress', campaignId: activeCampaignGeneration }));
    return;
  }

  // Same campaign pre-creation logic as CLI path
  // (channels, creation list, DB records — reuse the existing code)
  // ... parse channels, build creation list, create campaign records ...

  activeCampaignGeneration = campaignId;

  // SSE headers
  res.writeHead(200, { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache', 'Connection': 'keep-alive' });
  (res as any).flushHeaders?.();
  res.write(`data: ${JSON.stringify({ type: 'session', campaignId, creationCount: creationSlideIterMap.length })}\n\n`);

  // Track completion
  let apiCompletedCount = 0;
  const apiTotalCount = creationSlideIterMap.length;

  // Run N parallel API pipelines
  for (const entry of creationSlideIterMap) {
    const pipelineCtx: PipelineContext = {
      prompt: prompt || 'Generate a marketing creation',
      creationType: entry.creation.creationType,
      workingDir: path.join(path.dirname(entry.absHtmlPath), 'working'),
      htmlOutputPath: entry.absHtmlPath,
      creationId: entry.creation.id,
      campaignId,
    };

    // Fire and forget — each pipeline runs independently
    runApiPipeline(pipelineCtx, res)
      .then(() => {
        try { updateIterationGenerationStatus(entry.iterationId, 'complete'); } catch {}
      })
      .catch((err) => {
        try { res.write(`event: stderr\ndata: ${JSON.stringify({ creationId: entry.creation.id, text: `Pipeline error: ${err.message}` })}\n\n`); } catch {}
      })
      .finally(() => {
        apiCompletedCount++;
        if (apiCompletedCount >= apiTotalCount) {
          activeCampaignGeneration = null;
          try {
            res.write(`event: done\ndata: ${JSON.stringify({ code: 0, campaignId })}\n\n`);
            res.end();
          } catch {}
        }
      });
  }

  // Client disconnect cleanup
  req.on('close', () => { activeCampaignGeneration = null; });
}
```

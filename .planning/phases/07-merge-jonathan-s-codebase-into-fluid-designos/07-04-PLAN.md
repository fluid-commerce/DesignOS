  <files>canvas/src/store/editor.ts, canvas/src/components/SlotField.tsx, canvas/src/components/ContentEditor.tsx</files>
  <action>
    Create `canvas/src/store/editor.ts` (Zustand store):
    - selectedIterationId: string | null
    - slotSchema: SlotSchema | null (loaded from iteration's slot_schema)
    - slotValues: Record<string, string> (current values keyed by selector)
    - isDirty: boolean (tracks unsaved changes)
    - selectIteration(id): fetches iteration data from API, loads schema, extracts current values
    - updateSlotValue(sel, value): updates local state + sends postMessage to iframe
    - saveUserState(): PATCHes /api/iterations/:id/user-state with current slotValues
    - iframeRef: stored reference to the active iframe for postMessage
    - clearSelection(): resets all state

    Create `canvas/src/components/SlotField.tsx`:
    - Renders the correct input component based on SlotField type:
      - TextField: textarea (rows from field config, or auto-detect). Mode controls how value is applied:
        - 'text': plain textContent replacement
        - 'pre': preserves whitespace (white-space: pre-wrap)
        - 'br': converts newlines to <br> tags
      - ImageField: image preview + "Browse" button placeholder (DAM integration is scaffolded but not functional). Shows dims hint. Clicking opens PhotoReposition.
      - DividerField: visual separator with label text (carousel section headers)
    - Each field change calls editor store's updateSlotValue, which sends postMessage to iframe
    - Fields are labeled with field.label

    Create `canvas/src/components/ContentEditor.tsx`:
    - Right sidebar container component
    - Header with iteration info (source: AI/template, status badge, iteration number)
    - Renders SlotField for each field in slotSchema.fields
    - If slotSchema is null (no schema emitted by AI): shows graceful fallback "No editable slots available"
    - If slotSchema.brush is set: shows BrushTransform section (Task 2)
    - If slotSchema.carouselCount > 1: shows CarouselSelector (Task 2)
    - Export section at bottom (Task 2)
    - Save button that calls saveUserState() when isDirty
    - Listens for postMessage responses from iframe (for value extraction on first load)
    - On mount, sends a 'readValues' postMessage to iframe to extract current values for all selectors in schema
  </action>

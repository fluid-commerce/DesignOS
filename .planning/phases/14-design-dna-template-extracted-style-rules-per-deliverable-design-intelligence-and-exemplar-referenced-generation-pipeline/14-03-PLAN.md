Create a new component (either inline in the existing Templates viewport file or as a separate file if needed) called `DesignDnaPanel`. This component:

1. **Fetches design rules** on mount via `fetch('/api/design-rules')`, parses JSON into an array of DesignRule objects (matching the interface: id, scope, platform, archetypeSlug, label, content, sortOrder, updatedAt).

2. **Renders rules grouped by scope** in this order:
   - **General Social Media** heading (scope === 'global-social') — 1 rule
   - **Instagram** heading (scope === 'platform', platform === 'instagram') — 1 rule
   - **LinkedIn** heading (scope === 'platform', platform === 'linkedin') — 1 rule
   - **Archetype Design Notes** heading (scope === 'archetype') — 7 rules, each with its label (e.g., "Problem-First", "Quote/Testimonial")

3. **Each rule renders as:**
   - Label in bold (h3-style, color #fff, fontSize 0.95rem)
   - Content displayed as a `<textarea>` (read-only by default, editable on click)
   - Styling: dark background (#1a1a1a), border 1px solid #333, border-radius 6px, padding 12px, font-family monospace, color #ccc, width 100%, min-height 120px for general/platform rules, min-height 80px for archetype notes
   - On click: border changes to #44B2FF (blue highlight), becomes editable
   - On blur or Ctrl+Enter: saves via `fetch('/api/design-rules/${rule.id}', { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ content: newContent }) })`, reverts border to #333
   - Show a small "Saved" indicator (green text, fades after 2 seconds) on successful save

4. **Tab integration**: The Templates viewport (wherever it lives) should have a tab bar at the top with at minimum:
   - "Templates" tab (existing template gallery content)
   - "Social Media DNA" tab (the new DesignDnaPanel)

   Use useState for activeTab. Tab bar styling: flex row, each tab is a button with border-bottom: 2px solid transparent when inactive, border-bottom: 2px solid #44B2FF when active. Background transparent, color #888 inactive, #fff active. No extra wrapper divs — keep it minimal.

5. **Overall container styling**: padding 1rem, overflow-y auto, height 100%. Dark theme consistent with the rest of the app.

Follow the VoiceGuide.tsx pattern for the inline editing UX (textarea, onBlur save, editing state tracking). Use useState hooks for: rules array, editingId (which rule is being edited), editContent (current textarea value), savedId (for flash "Saved" indicator).

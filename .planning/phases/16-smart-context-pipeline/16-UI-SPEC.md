## Typography

All sizes and weights are pre-established in the codebase. This contract enforces the existing scale — no new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 | 1.5 |
| Label / caption | 11.5px (0.72rem) | 400 | 1.4 |
| Section heading | 16px (1rem) | 600 | 1.2 |
| Page heading | 20px (1.25rem) | 600 | 1.2 |

Source: `canvas/src/components/StreamMessage.tsx`, `canvas/src/components/TemplatesScreen.tsx`, `canvas/src/components/VoiceGuide.tsx`

Notes:
- Body at 0.875rem / 1.5 matches existing StreamMessage and TemplatesScreen body text.
- Label at 0.72rem matches existing stage badges and tab labels. Differentiated from body by size (11.5px vs 14px) and color (`--text-secondary` #888 vs `--text-primary` #e0e0e0) — weight stays 400 for both.
- Section headings at 1rem / 600 matches VoiceGuide h4 and inline group headers across the app.
- No bold display text needed — Settings page is a data editor, not a marketing surface.
- Two weights only: 400 (body + label) and 600 (section heading + page heading).

## Component Inventory

### New: SettingsScreen (viewport, new NavTab `settings`)

Full-viewport page mounted via AppShell renderViewport switch. Contains:
- Page heading + subtitle
- Context map editor section

Layout: single-column scrollable, max-width 900px centered within available viewport width.

### New: ContextMapEditor

Table/grid UI showing all `context_map` entries. Each row: creation type, stage, sections (comma-separated slugs or wildcards), priority (1-100), max tokens (optional number).

Inline editing: clicking any cell activates an input/textarea matching TemplatesScreen DesignDnaPanel pattern — click to edit, blur or Cmd+Enter to save, Escape to cancel. Optimistic update on save; revert on error.

Row actions: delete icon (trash, 16x16 inline SVG, `aria-label="Delete entry"`) appearing on row hover. Delete triggers a single-line confirm strip below the row (not a modal): "Remove this entry? [Remove entry] [Keep entry]".

Add row: "+ Add entry" button at table bottom. Inserts blank row in editing state. Creation type and stage are select dropdowns (fixed option sets). Sections is a text input accepting comma-separated slugs or `category:*` wildcards.

### New: ContextPanel (in ChatSidebar, per generation stage)

Collapsible section appearing below each stage-running/stage-done badge in the chat sidebar stream. Triggered by new SSE event type `context-injected`.

Collapsed state: one-line pill — "Context · {N} sections" in label color (#888), right-chevron.
Expanded state: sections list (slug chips, #1e2d40 background, #7bb8e0 text matching existing tool-start pill style), token estimate line, optional gap signal line in orange.

Chevron toggle: 14x14 inline SVG, same as existing AppShell ChevronIcon. `aria-label="Expand context details"` when collapsed; `aria-label="Collapse context details"` when expanded.

### Modified: LeftNav

Add `settings` tab with a gear/settings icon (20x20 inline SVG, strokeWidth 1.75). Positioned at bottom of the main nav list, above the AI Chat toggle. Label: "Settings".

### Modified: AppShell (renderViewport)

Add `case 'settings'` returning `<SettingsScreen />`.

## Interaction Patterns

### Context map grid — inline editing
- Click any cell to enter edit mode (same row, same cell)
- Edit state: input or textarea replaces static text, border-color transitions to `--accent` (#44B2FF)
- Blur saves (optimistic), reverts on API error
- Cmd+Enter saves immediately (keyboard shortcut matches TemplatesScreen convention)
- Escape cancels without saving
- "Saved" flash appears in-cell for 2000ms, color #7be0a0 (matches existing tool-done green)

### Context panel — expand / collapse
- Default: collapsed after stage completes
- Click anywhere on the collapsed pill to expand
- Expand/collapse uses height animation: `max-height` 0 → auto with `overflow: hidden`, transition 0.2s ease (matches ChatSidebar width transition convention)
- Gap signal line only shown when fallback tool calls occurred in that stage

### Context map — delete
- Trash icon appears on row hover (opacity 0 → 1 on hover)
- Click trash shows inline confirm strip; no full modal
- Confirm strip uses same background as row (#1a1a1e) with destructive text (#e08080) and two buttons: "Remove entry" (destructive) and "Keep entry" (cancel)

### Settings nav item
- Follows existing LeftNav pattern: 40x40 button, `--bg-card` background when active, `--accent` left border (2px) when active
- Tooltip on hover: "Settings" (same tooltip positioning fix needed — existing hardcoded tooltip positioning is a known pending todo)

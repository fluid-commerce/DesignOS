**D. SettingsScreen.tsx — Create context map editor**:

Full-viewport component. Follow the UI-SPEC exactly:
- Page heading: "Pipeline Settings" (20px, weight 600, color #fff)
- Page subtitle: "Configure which brand context each pipeline stage receives." (14px, color #888)
- Section heading: "Context Map" (16px, weight 600, color #fff)
- Section subtitle: "Maps (creation type, stage) to brand sections. Wildcards like `design-tokens:*` expand at runtime." (11.5px / 0.72rem, color #888)
- Max-width: 900px, centered, padding 48px top, 24px sides

Table columns: Creation Type, Stage, Sections, Priority, Token Budget
- Header row: fontSize 0.72rem, fontWeight 600, color #888, textTransform uppercase, letterSpacing 0.04em
- Data cells: backgroundColor #1a1a1e, padding 8px, fontSize 0.875rem (14px), color #e0e0e0

Inline editing pattern (from TemplatesScreen):
- State: `editingId: string | null`, `editField: string | null`, `editValue: string`, `savedId: string | null`
- Click cell -> set editingId + editField + editValue
- Blur -> save (PUT /api/context-map/:id), optimistic update, revert on error
- Cmd+Enter (onKeyDown e.metaKey && e.key === 'Enter') -> save immediately
- Escape -> cancel (set editingId null)
- savedId flash: set savedId on save success, clear after 2000ms via setTimeout

Creation Type and Stage: use `<select>` dropdowns in edit mode
- Creation type options: `instagram`, `linkedin`, `one-pager`
- Stage options: `copy`, `layout`, `styling`, `spec-check`

Sections: textarea in edit mode, comma-separated slugs/wildcards
Priority: number input (1-100)
Token Budget: number input (optional, empty = no limit)

"+ Add entry" button at bottom: inserts a blank row in editing state with POST /api/context-map on first save.

Delete: trash icon (16x16 inline SVG, opacity 0 on default, 1 on row hover). Click shows inline confirm strip below row: "Remove this entry? This stage will fall back to agent self-discovery." with [Remove entry] (color #e08080) and [Keep entry] (color #888) buttons. Confirm calls DELETE /api/context-map/:id.

Empty state: "No context map entries" heading, "Add entries to define what brand context each stage receives. Without a map, agents self-discover context using DB tools." body in #555.

Fetch data on mount: `useEffect(() => { fetch('/api/context-map').then(r => r.json()).then(setEntries) }, [])`.

Error states from UI-SPEC:
- Load failure: "Could not load context map. Check the server is running."
- Save failure: "Save failed — changes were reverted. Try again."

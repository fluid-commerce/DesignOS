## What Was Built

### Task 1: Server Infrastructure

**DB Schema (canvas/src/lib/db.ts):**
- Added `brand_assets` table: id, name, category, file_path UNIQUE, mime_type, size_bytes, tags, created_at
- Added `campaign_assets` table: id, campaign_id FK, file_path, url_path, mime_type, size_bytes, created_at

**Asset Scanner (canvas/src/server/asset-scanner.ts):**
- New file exporting `scanAndSeedBrandAssets(assetsDir: string)`
- Reads top-level subdirs as categories (brushstrokes, fonts, circles, etc.)
- Inserts with `INSERT OR IGNORE` for idempotent rescans
- MIME type lookup from extension, skips hidden files and non-file entries
- Called non-blocking on Vite `configureServer` startup

**DB API (canvas/src/server/db-api.ts):**
- Added `BrandAsset` interface with `url: string` field set to `/fluid-assets/{file_path}`
- Added `getBrandAssets(category?: string)` — full catalog or filtered by category

**Watcher Middleware (canvas/src/server/watcher.ts):**
- Added `GET /api/brand-assets?category=` endpoint before existing `/api/assets` route
- Added `/fluid-campaigns/` middleware serving `.fluid/campaigns/{cId}/...` with path traversal guard (`campaignsBase + path.sep`)
- Replaced `?download=1` base64 block with `?export=zip` using archiver: rewrites `/fluid-assets/` to `assets/` in HTML, bundles referenced files into ZIP

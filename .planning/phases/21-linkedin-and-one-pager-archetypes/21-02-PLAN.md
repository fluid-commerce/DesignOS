  <action>
Create 6 LinkedIn archetype directories. Each must follow SPEC.md exactly. All archetypes use 1200x627 dimensions, `font-family: sans-serif`, `background: #111`, no brand assets, no brand fonts, no side labels, no uppercase transforms.

**For each adapted archetype:** Read the Instagram source archetype first. Restructure for landscape canvas — wider horizontal zones, less vertical stacking, larger text for wider reading. LinkedIn posts are wider (1200px) and shorter (627px vs 1080px) — this means more horizontal flow, less vertical space.

**1. hero-stat-li** (adapted from hero-stat):
- Landscape version of the stat-hero pattern: eyebrow + headline on left side, 3-stat row across bottom or right zone
- Key structural change: stats can spread horizontally across the full 1200px width instead of stacking vertically
- Fields: eyebrow, headline, body-copy, stat-1-num, stat-1-label, stat-2-num, stat-2-label, stat-3-num, stat-3-label

**2. data-dashboard-li** (adapted from data-dashboard):
- 4 stat columns across full width instead of 2x2 grid — natural landscape fit
- Fields: eyebrow, headline, stat-1-value, stat-1-label, stat-2-value, stat-2-label, stat-3-value, stat-3-label, stat-4-value, stat-4-label, footer-cta

**3. split-photo-text-li** (adapted from split-photo-text):
- Left/right split is the natural LinkedIn pattern — image placeholder on left, text content on right (or vice versa)
- Key structural advantage: landscape aspect ratio means the 50/50 split gives each side meaningful dimensions
- Image placeholder uses 1x1 base64 data URI per SPEC.md
- Fields: headline, body-copy, image (type: image with `data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNk+M9QDwADhgGAWjR9awAAAABJRU5ErkJggg==` src)

**4. minimal-statement-li** (adapted from minimal-statement):
- Wide centered text — one bold statement across the full 1200px landscape
- More horizontal breathing room than 1080x1080 version
- Fields: headline, supporting-text

**5. quote-testimonial-li** (adapted from quote-testimonial):
- LinkedIn testimonial culture — portrait/avatar left, quote text right
- Landscape gives generous text width alongside portrait area
- Image placeholder for portrait, using same base64 1px PNG data URI
- Fields: quote-text, attribution-name, attribution-title, portrait-image (type: image)

**6. article-preview-li** (LinkedIn-only, no Instagram equivalent):
- Article card layout: large title area, 2-3 line excerpt, author name, category/topic label
- Mimics the LinkedIn article share card format — clean, editorial, professional
- Fields: article-title, article-excerpt, author-name, category-label

**For EVERY archetype:**
- HTML comment: `<!-- archetype: {slug} | target: 1200x627 -->`
- Body: `width: 1200px; height: 627px; overflow: hidden; position: relative; margin: 0; padding: 0; font-family: sans-serif; background: #111;`
- `<div class="decorative-zone">` as first child of body
- `<!-- SLOT: {class-name} -->` comment before each content element
- All CSS in single `<style>` block — NO inline styles
- Text colors: `#ffffff` for primary, `rgba(255,255,255,0.5)` for muted/secondary
- Placeholder text in ALL elements — never empty divs
- Reset: `* { box-sizing: border-box; margin: 0; padding: 0; }`

**schema.json for each:**
- `"archetypeId": "{slug}"` (e.g. `"hero-stat-li"`)
- `"platform": "linkedin-landscape"`
- `"width": 1200, "height": 627`
- `"brush": null, "brushAdditional": []`
- Field `sel` values must use single class selectors (`.headline`, not `.zone .headline`)
- Every field `sel` class must exist in index.html

**README.md for each:**
Following the same structure as existing archetypes (see hero-stat/README.md):
1. What it is (one sentence)
2. Structural pattern
3. Content type fit
4. When to use
5. When NOT to use

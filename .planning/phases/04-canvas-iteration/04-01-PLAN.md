Build the server-side infrastructure and data layer:

1. **Filesystem watcher Vite plugin (canvas/src/server/watcher.ts):**
   - Export `fluidWatcherPlugin(workingDir: string): Plugin`
   - Uses chokidar v4 to watch the workingDir path
   - On any file change event, sends `fluid:file-change` custom HMR event via `server.ws.send()`
   - Debounce events by 300ms to handle rapid multi-file writes
   - `ignoreInitial: true`, depth 3

2. **Update vite.config.ts:**
   - Import and use fluidWatcherPlugin with path `../.fluid/working` (relative to canvas/)
   - Add API proxy or custom middleware to serve session data: GET /api/sessions returns discovered sessions, GET /api/sessions/:id returns session detail with file contents

3. **Session discovery (canvas/src/lib/sessions.ts):**
   - `discoverSessions(workingDir)`: Reads .fluid/working/, finds session directories (pattern: YYYYMMDD-HHMMSS), reads lineage.json from each, returns SessionSummary[] sorted newest-first
   - `loadSession(workingDir, sessionId)`: Reads all HTML files in session directory, reads lineage.json, reads annotations.json if exists, returns full session data
   - Handle both Phase 2 (`entries[]`) and Phase 4 (`rounds[]`) lineage formats gracefully
   - Use `countVariations()` helper that looks for styled.html files in subdirectories or root

4. **Zustand session store (canvas/src/store/sessions.ts):**
   - State: `sessions: SessionSummary[]`, `activeSessionId: string | null`, `activeSessionData: SessionData | null`
   - Actions: `refreshSessions()`, `selectSession(id)`, `clearSelection()`
   - `refreshSessions` calls the /api/sessions endpoint
   - `selectSession` fetches full session data including HTML file contents

5. **File watcher hook (canvas/src/hooks/useFileWatcher.ts):**
   - Listens for `fluid:file-change` HMR custom event via `import.meta.hot.on()`
   - On change, calls `refreshSessions()` from the zustand store
   - Debounce to avoid excessive re-renders

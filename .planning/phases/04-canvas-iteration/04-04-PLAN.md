  <action>
Create the /fluid-design-OS launcher and supporting scripts:

1. **Start script (canvas/scripts/start.sh):**
   - Check if canvas dev server is already running (test -f canvas/.pid && kill -0)
   - If not running: cd canvas && npm run dev > /dev/null 2>&1 & echo $! > .pid
   - Wait for server to be ready (curl loop on localhost:5174, max 10 seconds)
   - Print the URL
   - Make executable: chmod +x

2. **Stop script (canvas/scripts/stop.sh):**
   - Read PID from canvas/.pid
   - Kill the process and its children
   - Remove .pid file
   - Print confirmation
   - Make executable: chmod +x

3. **Skill definition (.claude/skills/fluid-design-os/SKILL.md):**
   Follow existing skill patterns (fluid-social, etc.):
   ```yaml
   ---
   name: fluid-design-os
   description: "Start/stop the Fluid Design OS canvas for viewing, annotating, and iterating on generated assets."
   context: fork
   argument-hint: '[stop]'
   allowed-tools: Bash, Read
   ---
   ```

   Skill behavior:
   - If $ARGUMENTS contains "stop": Run the stop script, print "Canvas stopped."
   - Otherwise (start):
     - Check if canvas/node_modules exists; if not, run `cd canvas && npm install`
     - Run the start script
     - Print instructions: canvas URL, how to generate assets (/fluid-social --variations 3), how to stop (/fluid-design-OS stop)
     - Document MCP server lifecycle: "The MCP agent bridge (fluid-canvas) is auto-managed by Claude Code via .mcp.json. When Claude Code needs to use MCP tools (push_asset, read_annotations, etc.), it spawns the MCP server automatically. No manual MCP process management is needed."
     - Note: start.sh manages only the Vite dev server. The MCP server is a separate stdio process managed by Claude Code's MCP infrastructure (configured in .mcp.json at project root, created in Plan 02).

4. **Update sync.sh:**
   - Add fluid-design-os to the list of skills distributed to ~/.claude/commands/ and ~/.cursor/skills/
   - Follow existing pattern for skill distribution (symlink or copy)

5. **Add canvas/.pid and canvas/node_modules to .gitignore** if not already present.

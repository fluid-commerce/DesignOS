Create `tools/validate-archetypes.cjs` following the `brand-compliance.cjs` pattern (CJS, shebang, dual output).

**Constants:**
- `ARCHETYPES_DIR = path.resolve(__dirname, '../archetypes')`
- `REQUIRED_FILES = ['index.html', 'schema.json', 'README.md']`
- `REQUIRED_DIMS = { width: 1080, height: 1080 }` (Instagram archetypes)
- `KNOWN_FIELD_TYPES = ['text', 'image', 'divider']`
- `KNOWN_TEXT_MODES = ['text', 'pre', 'br']`

**Usage:** `node tools/validate-archetypes.cjs [slug|"all"]`
- No args or `all`: validate all subdirectories in `archetypes/` (skip `components/`)
- Specific slug: validate only `archetypes/{slug}/`

**Checks per archetype (in `validateArchetype(dir)` function):**
1. All `REQUIRED_FILES` exist in directory
2. `schema.json` parses as valid JSON
3. `schema.json` has `width` (number), `height` (number), `fields` (array)
4. `width === 1080` and `height === 1080`
5. `brush` is `null` or absent (not a non-null string)
6. `brushAdditional` is `[]` or absent (not a non-empty array)
7. No `templateId` field present
8. Every `fields[]` entry has valid `type` in KNOWN_FIELD_TYPES
9. Every text field has `sel` (string), `label` (string), `mode` in KNOWN_TEXT_MODES
10. Every image field has `sel` (string), `label` (string)
11. Every `fields[].sel` value: extract the class name (e.g. `.headline` -> `headline`, `.photo img` -> `photo`), check that it appears in `index.html` content via `html.includes(className)`

**Output format:**
- stdout: JSON array of violation objects `{ severity: 'error'|'warning', code: string, slug: string, message: string }`
- stderr: Human-readable summary with counts (e.g. `hero-stat: 0 errors, 0 warnings`)
- Exit code: 1 if any `severity: 'error'`, 0 otherwise

**Edge cases:**
- If `archetypes/` directory does not exist, write `[]` to stdout, "No archetypes directory found" to stderr, exit 0
- If specific slug directory does not exist, report `MISSING_DIR` error for that slug
- If no subdirectories found (empty archetypes/), write `[]` to stdout, exit 0

**Selector extraction logic for check 11:**
```javascript
// For sel like ".headline" -> check html includes 'class="headline"' OR 'class="' contains 'headline'
// For sel like ".photo img" -> check html includes 'class="photo"' OR class contains 'photo'
// For sel like ".stat-1-num" -> check html includes 'stat-1-num'
// For sel like ".category span" -> check html includes 'category'
// Simple: extract first segment, strip leading dot, check html.includes(className)
function extractClassName(sel) {
  return sel.split(/[\s>+~]/)[0].replace(/^\./, '');
}
```

```markdown
# Archetype System

Brandless content layout skeletons for social media posts. Each archetype defines
a content structure (HTML), editable slot schema (JSON), and documentation (README).

## Directory Structure

archetypes/
  components/README.md   — Reusable HTML/CSS component patterns
  {slug}/
    index.html           — 1080x1080 brandless skeleton with placeholder content
    schema.json          — SlotSchema (width, height, fields, brush: null)
    README.md            — Purpose, slots, when to use

## Instagram Archetypes (1080x1080)

| Slug | Description |
|------|-------------|
| hero-stat | Giant stat number + supporting text |
| photo-bg-overlay | Full-bleed photo with text overlay |
| split-photo-text | 50/50 photo left, text right |
| quote-testimonial | Quote with portrait + attribution |
| minimal-statement | Single bold statement, maximum whitespace |
| data-dashboard | 3-stat grid with headline and footnote |

## Conventions

- `body` is the 1080x1080 canvas (position: relative)
- All slot elements use position: absolute
- CSS class names match `sel` values in schema.json
- Placeholder content is visible wireframe text (not empty)
- `brush: null` and `brushAdditional: []` — brand decorative layer added at generation time
- No `templateId` in schema.json — archetypes are not templates
- No brand asset URLs — archetypes are brandless
- `<div class="decorative-zone">` placeholder for brand injection

## Validation

Run `node tools/validate-archetypes.cjs` to check all archetypes.
Run `node tools/validate-archetypes.cjs {slug}` to check one.
```

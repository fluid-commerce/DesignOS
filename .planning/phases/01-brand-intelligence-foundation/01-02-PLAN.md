    **tools/compile-rules.cjs** — Reads brand docs, extracts machine-checkable rules into rules.json:
    - Parse brand/design-tokens.md for: allowed hex colors (separate social vs website arrays), allowed font families (all aliases), spacing values, opacity ranges, border radius values
    - Parse brand/social-post-specs.md for: target dimensions, background color rules, footer structure requirements
    - Parse brand/website-section-specs.md for: schema counts (13 font sizes, 13 colors, 5 weights, 7 button settings, section/container settings)
    - Extract Weight: N annotations from parsed rules
    - Write tools/rules.json with structure from RESEARCH.md (colors.allowed_hex, colors.rules, fonts.allowed_families, fonts.rules, spacing.rules, schema counts, dimensions)
    - Include version field and compiled timestamp
    - Usage: `node tools/compile-rules.cjs` (no args, reads from brand/ dir relative to script location)

    **tools/rules.json** — Generated by compile-rules.cjs. Structure:
    ```json
    {
      "version": "1.0.0",
      "compiled": "ISO timestamp",
      "colors": {
        "social": { "allowed_hex": [...], "background": "#000000" },
        "website": { "allowed_hex": [...] },
        "rules": [{ "id": "...", "pattern": "regex", "message": "...", "weight": N, "context": "social|website|all" }]
      },
      "fonts": {
        "allowed_families": ["NeueHaasDisplay", "NeueHaas", "FLFont", "flfontbold", "Inter"],
        "rules": [...]
      },
      "spacing": { "rules": [...] },
      "schema": {
        "font_size_count": 13, "color_count": 13, "weight_count": 5,
        "button_settings": ["show", "text", "url", "style", "color", "size", "weight"],
        "section_settings": [...], "container_settings": [...]
      },
      "dimensions": {
        "instagram": { "width": 1080, "height": 1080 },
        "linkedin_landscape": { "width": 1200, "height": 627 },
        "linkedin_tall": { "width": 1340, "height": 630 }
      },
      "thresholds": { "error": 81, "warning": 51, "info": 21, "hint": 1 }
    }
    ```

    **tools/brand-compliance.cjs** (CLI-02) — Validates HTML/CSS against brand tokens:
    - Accepts file path as arg: `node tools/brand-compliance.cjs path/to/file.html`
    - Optional `--context social|website` flag to scope rules (defaults to detecting from file content or using "all")
    - Reads rules.json (not brand markdown at runtime)
    - Checks: hex colors against allowed list, font-family against allowed families, rgba patterns, hardcoded spacing values
    - Per-violation output: { file, line, column, rule, severity, weight, message }
    - Severity derived from weight: >=81 error, >=51 warning, >=21 info, else hint
    - Exit code: 1 if any errors (weight >= 81), 0 otherwise
    - Human summary to stderr: "X errors, Y warnings, Z info" with color-coded output if terminal supports it
    - Follow the code example pattern from RESEARCH.md (hex color matching with normalization, font family extraction)

    **tools/schema-validation.cjs** (CLI-01) — Validates .liquid schema completeness:
    - Accepts file path: `node tools/schema-validation.cjs path/to/file.liquid`
    - Extracts {% schema %}...{% endschema %} block and parses as JSON
    - Counts select-type settings options per category: font sizes, colors, weights
    - Reports shortfalls: "Font sizes: 10/13 (missing 3)" with specific missing values if detectable
    - Checks button settings completeness (7 required per button)
    - Checks section/container settings presence
    - Exit code: 1 if any required counts not met, 0 if passing
    - IMPORTANT: Parse JSON programmatically after extraction, do NOT regex-count options (per RESEARCH.md pitfall #6)

    **tools/dimension-check.cjs** (CLI-03) — Validates HTML asset dimensions:
    - Accepts file path: `node tools/dimension-check.cjs path/to/file.html`
    - Optional `--target instagram|linkedin_landscape|linkedin_tall|WIDTHxHEIGHT`
    - Extracts width/height from: inline styles on body/container, CSS width/height properties, meta viewport
    - Compares against target dimensions from rules.json
    - Auto-detects target if not specified (looks for dimension hints in HTML comments or container sizing)
    - Reports: expected WxH, found WxH, match/mismatch

    **tools/scaffold.cjs** (CLI-04) — Generates Gold Standard .liquid skeleton:
    - Usage: `node tools/scaffold.cjs section-name [--output path]`
    - Generates a .liquid file with:
      * Full schema pre-filled with all required settings (13 font sizes, 13 colors, 5 weights per text element, 7 button settings, section/container settings)
      * Content slot markers: `<!-- SLOT: headline -->`, `<!-- SLOT: body_copy -->`, etc.
      * Template structure matching Gold Standard workflow
      * CSS using variables/utility classes (no hardcoded values)
    - The generated skeleton MUST pass schema-validation.cjs with zero errors
    - Creates templates/gold-standard/ directory if it doesn't exist

    **tools/validate-on-write.sh** — PostToolUse hook wrapper:
    - Reads JSON from stdin (tool_input with file_path)
    - Routes .liquid files to schema-validation.cjs
    - Routes .html files to brand-compliance.cjs
    - Passes through the file path as argument
    - Make executable (chmod +x)

    After creating all tools, run compile-rules.cjs to generate the initial rules.json.

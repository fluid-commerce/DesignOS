  <action>
    **Read ~/.agents/sync.sh first** to understand the existing pattern. The project's sync.sh follows the same architecture but is PROJECT-specific (does NOT overwrite global ~/.agents/ skills).

    **sync.sh** — Distribution script:
    - Shebang: #!/bin/bash
    - set -euo pipefail
    - Detect repo root (directory where sync.sh lives)
    - Accept optional --target flag: `claude`, `cursor`, `both` (default: both)

    For Claude Code distribution:
    - Primary method: Add repo as --add-dir in Claude Code (skills in .claude/skills/ auto-discovered with hot reload)
    - Fallback: Symlink project skills into ~/.claude/skills/ with `fluid-` namespace prefix to avoid conflicts
      * ~/.claude/skills/fluid-brand-intelligence/ -> {repo}/.claude/skills/brand-intelligence/
      * ~/.claude/skills/fluid-brand-compliance-check/ -> {repo}/.claude/skills/brand-compliance-check/
      * ~/.claude/skills/fluid-scaffold-section/ -> {repo}/.claude/skills/scaffold-section/
    - Copy .claude/settings.json hooks to project scope (NOT global — these hooks only make sense in the project context)
    - DO NOT touch ~/.agents/ or ~/.claude/commands/ (those belong to the user's global system)

    For Cursor distribution:
    - Copy skill files to ~/.cursor/skills/fluid-brand-intelligence/ etc. (with fluid- prefix)
    - Adapt frontmatter if needed for Cursor format

    Features:
    - Idempotent: re-running produces identical results, overwrites stale symlinks
    - --dry-run flag: show what would happen without doing it
    - --uninstall flag: remove all fluid-* symlinks/copies
    - Print summary of what was installed/updated
    - Check that brand/ and tools/ directories exist before running (fail with helpful message if not)
    - Make executable (chmod +x)

    **install.md** — Machine-readable installation instructions:
    - Written for AGENTS to follow autonomously (not just humans)
    - Step-by-step with exact commands:
      1. Clone the repo: `git clone <repo-url> ~/fluid-creative-os` (placeholder URL)
      2. Run sync: `cd ~/fluid-creative-os && ./sync.sh`
      3. Add as additional directory in Claude Code: `claude --add-dir ~/fluid-creative-os`
      4. Verify: `ls ~/.claude/skills/ | grep fluid` should show 3 skills
      5. Test: In any project, type `/brand-compliance-check` — should be recognized
    - Troubleshooting section: common issues (permissions, existing conflicts, path issues)
    - Uninstall: `./sync.sh --uninstall`
    - Each step marked with expected output so the agent can verify success
    - Format: numbered steps with `EXPECTED:` annotations after each command
  </action>

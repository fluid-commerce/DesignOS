  <action>
Build each of the 11 section types following the exact same Gold Standard schema pattern proven in hero.liquid. Use `node tools/scaffold.cjs {section-name}` as starting point for each, then customize the template HTML for that section's layout and block types.

**Section types and their specific elements:**

| Section | Text Elements | Blocks | Special |
|---------|--------------|--------|---------|
| features-grid | heading, subheading | feature_item (icon, title, description -- each with 6 settings) | Grid layout, 3-column responsive |
| testimonials | heading | testimonial (quote, author, role -- each text with 6 settings, plus image_picker) | Carousel or grid |
| cta-banner | heading, body | none | Dual buttons (primary + secondary), full-width background |
| image-text | heading, body | none | Split layout (image_picker + text side), reversible via setting |
| statistics | heading | stat_item (value + label, each with 6 settings) | Large numbers, grid layout |
| faq-accordion | heading | faq_item (question + answer, each with 6 settings) | Details/summary HTML for accordion |
| logo-showcase | heading | logo_item (image_picker + alt text + link url) | Grid layout, grayscale option |
| pricing | heading, subheading | pricing_tier (name, price, period, features richtext, cta button -- each text with 6 settings) | Card layout, highlighted tier option |
| content-richtext | heading | none | richtext setting for body, WYSIWYG-friendly |
| video | heading, body | none | video_url setting (YouTube/Vimeo embed), overlay text |
| newsletter | heading, body | none | Form placeholder (email input + submit button), privacy text |

**For EVERY section:**
1. Start with `node tools/scaffold.cjs {name}` to get correct schema skeleton
2. Schema setting order: Content -> Interactive -> Layout -> Container (Gold Standard)
3. Every text element: 6 settings (family, size mobile, size desktop, weight, color, content)
4. Every button: 7 settings (show, text, url, style, color, size, weight) using btn utility class pattern
5. Section settings: 5 (background_color, background_image, padding_y_mobile, padding_y_desktop, border_radius)
6. Container settings: 7 (all paddings, background, radius)
7. Block elements: `{{ block.fluid_attributes }}` on outermost container
8. FIXED/FLEXIBLE/OPTIONAL annotations as HTML comments
9. All styles via utility classes from schema settings -- ZERO hard-coded values
10. Validate: `node tools/schema-validation.cjs templates/sections/{name}.liquid` must pass

**Block schema pattern (for sections with blocks):**
```json
"blocks": [
  {
    "type": "feature_item",
    "name": "Feature Item",
    "settings": [
      { "type": "text", "id": "title", "label": "Title", "default": "Feature Title" },
      { "type": "select", "id": "title_font_family", ... },
      // ... all 6 settings per text element
    ]
  }
]
```

**After all 11 are built, batch validate:**
```bash
for f in templates/sections/*.liquid; do node tools/schema-validation.cjs "$f"; done
```
All must pass with 0 errors.

**Part B: /fluid-theme-section orchestrator (.claude/skills/fluid-theme-section/SKILL.md)**

Mirror the `/fluid-social` orchestrator pattern but for mode=section. Key differences:

**Frontmatter:**
```yaml
name: fluid-theme-section
description: "Generate Gold Standard compliant .liquid website sections. Orchestrates copy, layout, styling, and spec-check agents to produce sections with complete schema settings."
context: fork
disable-model-invocation: true
argument-hint: '"section description" [--type hero|features-grid|testimonials|...] [--template name] [--debug]'
allowed-tools: Agent, Bash, Read, Write, Glob, Grep, Edit
```

**Argument parsing:**

| Flag | Values | Default | Purpose |
|------|--------|---------|---------|
| `--type` | section type name | (inferred from prompt) | Which section type to generate |
| `--template` | template name | (none) | Use specific template as starting point |
| `--debug` | (flag) | off | Preserve working directory |

**Pipeline:** Same 4-stage delegation (copy -> layout -> styling -> spec-check) but:
- Pass `mode=section, platform=shopify` to all agents
- Copy agent outputs section content (heading, subheading, body, button text)
- Layout agent outputs .liquid file with Gold Standard schema
- Styling agent ensures all utility classes are correctly applied (no hard-coded values)
- Spec-check runs schema-validation.cjs + brand-compliance.cjs

**Fix loop:** Same pattern as /fluid-social (severity-based routing, 3-iteration cap).

**Output:** Final .liquid file copied to `templates/sections/{type}.liquid` or a user-specified path.

**Session directory:** Same `.fluid/working/{sessionId}/` pattern with lineage.json.

**Part B: /fluid-one-pager orchestrator (.claude/skills/fluid-one-pager/SKILL.md)**

Mirror the `/fluid-social` orchestrator pattern but for mode=one-pager.

**Frontmatter:**
```yaml
name: fluid-one-pager
description: "Generate print-ready Fluid one-pager sales collateral from a simple prompt. Creates letter-size HTML/CSS with brushstrokes, side labels, and FLFont taglines."
context: fork
disable-model-invocation: true
argument-hint: '"topic or brief" [--type product-feature|partner|company|case-study|comparison] [--product connect|payments] [--template name] [--ref path] [--debug]'
allowed-tools: Agent, Bash, Read, Write, Glob, Grep, Edit
```

**Argument parsing:**

| Flag | Values | Default | Purpose |
|------|--------|---------|---------|
| `--type` | product-feature, partner, company, case-study, comparison | (inferred from prompt) | One-pager layout type |
| `--product` | connect, payments | (none) | Product context for copy agent |
| `--template` | template name | (none) | Use specific template as starting point |
| `--ref` | file path | (none) | Reference file for style matching |
| `--debug` | (flag) | off | Preserve working directory |

**Pipeline:** Same 4-stage delegation (copy -> layout -> styling -> spec-check) but:
- Pass `mode=one-pager` to all agents
- Copy agent outputs one-pager content zones (hero headline, stats, body grid, CTA)
- Layout agent outputs HTML structure with zone layout and @page rules
- Styling agent embeds fonts/assets, applies design tokens, adds brushstrokes
- Spec-check runs brand-compliance.cjs + dimension-check.cjs --target letter + holistic review (print layout, brand elements, content fit)

**Fix loop:** Same pattern as /fluid-social (severity-based routing, 3-iteration cap).

**Output:** Final HTML file with print-ready styling. Instruct user to open in browser and Print to PDF.

**Session directory:** Same `.fluid/working/{sessionId}/` pattern with lineage.json.
